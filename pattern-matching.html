Pattern Matching

Rust has no switch statement.


Instead, match is used.


Enforced exhaustive checking.


'_' is a catch-all, much like 'default'.


Multiple and range matching.


Unlike switch, match is an expression, not a statement.


Matching on Enums


-All possible values checked (no need for '_')


-Args are given on enum creation


Using match in place of switch

let x = 5;
let str = match x {
1 => println!("one"),
2 => println!("two"),
3 => println!("three"),
4 => println!("four"),
5 => println!("five"),
6 ... 9 => println!("six through nine"),
 0 | 10 => println!("zero or ten"),
 _ => println!("something else"),
 }
error: non-exhaustive patterns: `_` not covered Success!

Using match on enums

enum Program {
Quit,
ChangeColor(i32, i32, i32),
 ToRed(i32, i32, i32),
 ToBlue(i32, i32, i32),
 Move { x: i32, y: i32 },
Write(String),
}

 fn quit() { /* ... */ }
fn change_color(r: i32, g: i32, b: i32) { /* ... */ }
fn move_cursor(x: i32, y: i32) { /* ... */ }

fn run(prg: Program) {
match prg {
Program::Quit => quit(),
Program::ChangeColor(r, g, b) => change_color(r, g, b),
 Program::ToRed(r, g, b) => change_color(r, g, b),
 Program::ToBlue(r, g, b) => change_color(r, g, b),
 Program::Move { x: x, y: y } => move_cursor(x, y),
Program::Write(s) => println!("{}", s),
};
}
